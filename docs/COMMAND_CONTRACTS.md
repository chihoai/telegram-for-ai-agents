# Command Contracts (`--json`)

This document defines machine-readable payload contracts for local agent usage.
The canonical MCP tool schema export lives at `docs/tool-contracts.json` and is generated by `npm run export:contracts`.

General:

- Success: JSON object with `"ok": true`
- Failure: JSON object with `"ok": false` and `"error"` (non-zero exit code)
- Dates are ISO-8601 strings

## `tgchats whoami --json`

```json
{
  "ok": true,
  "account": {
    "displayName": "string",
    "id": 123,
    "username": "string|null"
  },
  "sessionPath": "string",
  "accountLabel": "string"
}
```

## `tgchats auth status --json`

```json
{
  "ok": true,
  "sessionPresent": true,
  "sessionPath": "string",
  "note": "string"
}
```

## `tgchats inbox --json`

```json
{
  "ok": true,
  "account": { "displayName": "string", "id": 123 },
  "count": 10,
  "warning": "string|null",
  "dialogs": [
    {
      "index": 1,
      "peer": {
        "id": 123,
        "type": "user|chat|channel",
        "displayName": "string",
        "username": "string|null"
      },
      "lastMessage": {
        "id": 1234,
        "date": "2026-03-03T12:00:00.000Z",
        "preview": "string"
      }
    }
  ]
}
```

Each dialog may also include an optional `peerRef` object when the runtime can provide a stable MCP-friendly peer descriptor:

```json
{
  "peerRef": {
    "id": "123",
    "kind": "user",
    "accessHash": "1234567890"
  }
}
```

## `tgchats open <peer> --json`

```json
{
  "ok": true,
  "peer": {
    "id": 123,
    "type": "user|chat|channel",
    "displayName": "string",
    "username": "string|null"
  },
  "metadata": {
    "tags": [{ "tag": "string", "source": "string" }],
    "company": { "companyName": "string", "role": "string|null" },
    "tasks": [
      {
        "taskId": 1,
        "dueAt": "2026-03-03T12:00:00.000Z",
        "why": "string",
        "priority": "low|med|high",
        "status": "open|done|snoozed"
      }
    ],
    "summary": {
      "content": "string",
      "updatedAt": "2026-03-03T12:00:00.000Z"
    }
  }
}
```

If DB is unavailable, `"metadata": null`.

## Additional JSON surfaces used by MCP

- `tgchats search ... --json`
- `tgchats folders list --json`
- `tgchats folders create|rename|delete|order|add|remove --json`
- `tgchats sync backfill --json`
- `tgchats sync once --json`
- `tgchats logout --json`

These follow the same top-level contract shape: success uses `"ok": true`, failure uses `"ok": false` with `"error"`.
